# task9 — 首页项目总览 + 最近项目列表最小版

## 目标
在当前已完成 task3-task8 的基础上，让用户回到首页后能看到“继续之前项目”的入口，而不只是静态介绍。

本次实现一个**最小可用的项目总览体验**：
- 首页增加“最近项目”区域
- 展示最近创建或更新的项目列表
- 每个项目卡片可直接进入对应工作区
- 首页保留原有静态介绍与路线入口，不做大改版

---

## 本次范围

### 需要完成
1. **新增项目列表读取能力**
   - 服务端获取最近项目
   - 可按 `updatedAt` 或 `createdAt` 倒序
2. **新增项目列表 API（建议做）**
   - `GET /api/projects`
   - 返回最近项目列表即可，不做分页
3. **首页增加最近项目区域**
   - 页面：`app/page.tsx`
   - 当有项目时展示最近项目卡片
   - 当没有项目时展示空状态引导
4. **项目卡片展示最少信息**
   - 项目名称
   - 当前状态
   - 创建时间或更新时间
   - 已完成任务进度（若可低成本复用）
   - 进入工作区按钮/链接
5. **基础异常处理**
   - 数据读取失败时要有温和降级，不要首页直接崩

### 不要做
- 不要做项目搜索
- 不要做分页
- 不要做项目删除
- 不要做项目编辑
- 不要做登录系统
- 不要做复杂筛选
- 不要做无关重构

---

## 数据与接口建议

### `GET /api/projects`
返回最近项目列表，数量控制在 6~10 个即可，建议默认 6 个。

返回结构示例：
```json
{
  "success": true,
  "projects": [
    {
      "id": "...",
      "title": "AI 简历优化助手",
      "status": "clarified",
      "createdAt": "...",
      "updatedAt": "...",
      "taskSummary": {
        "total": 8,
        "done": 3
      }
    }
  ]
}
```

如果你觉得 taskSummary 不适合放在 API，也可以只在服务端页面里组织，不强制。

---

## 页面要求

### 首页 `app/page.tsx`
在现有首页基础上增加一个“最近项目”区块。

建议结构：
- 顶部仍保留原有 hero / 入口内容
- 下方新增：`最近项目`
- 右侧或下方保留“新建项目”入口

#### 有项目时
展示项目卡片列表，每张卡片至少包含：
- 项目标题
- 状态文案
- 时间信息
- `继续进入工作区`

#### 无项目时
展示空状态：
- 文案示例：`你还没有创建项目，先从一个想法开始。`
- 提供按钮：`创建第一个项目`

#### 数据失败时
不要白屏。
可以：
- 静默 fallback 到空状态
- 或显示一行轻量提示

---

## 状态展示建议
统一复用已有项目状态，不新增复杂状态。

建议文案映射：
- `draft` → `刚创建`
- `clarifying` → `需求澄清中`
- `clarified` → `已完成澄清`

如果 task8 已有任务进度，可以补充类似：
- `3 / 8 个任务已完成`

但这不是强制，前提是改动成本低。
