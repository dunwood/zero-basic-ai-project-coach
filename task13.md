# task13 — 项目详情与轻量编辑页

## 目标
在当前已完成 task1 ~ task12 的基础上，补一个最小可用的“项目详情与轻量编辑”能力，让用户可以：

- 回看项目当前核心信息
- 修改项目标题和项目想法
- 快速回到澄清页继续补充
- 保持当前任务、设计书、review、workspace 主流程不被打乱

这次只做最小闭环，不扩展到复杂编辑器。

## API 约定

### `PATCH /api/projects/[id]`
请求体：
```json
{
  "title": "AI 简历优化助手",
  "idea": "我想做一个帮助用户梳理经历并生成简历初稿的工具。"
}
```

成功返回：
```json
{
  "success": true,
  "project": {
    "id": "...",
    "title": "...",
    "idea": "...",
    "status": "...",
    "updatedAt": "..."
  }
}
```

失败返回：
```json
{
  "success": false,
  "error": "错误信息"
}
```

### 校验规则
- `title` 必填，trim 后不能为空
- `idea` 必填，trim 后不能为空
- 保持简单，不要加复杂规则
