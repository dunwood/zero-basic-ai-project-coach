# task10 — 工作区总览升级 + 下一步引导闭环

## 目标
在已完成 task3 ~ task9 的基础上，继续完善工作区，让用户进入任意项目后，能更清楚地看到当前所处阶段、下一步该做什么、以及从哪里继续。

本次只做“工作区总览与引导闭环”，不做新的 AI 能力。

---

## 本次范围

### 需要完成
1. **工作区总览增强**
   - 页面：`/workspace/[id]`
   - 增加项目当前阶段总览
   - 增加“下一步操作”主按钮区域
   - 增加阶段导航卡片或阶段列表

2. **阶段判断逻辑**
   依据当前项目状态和已有数据，判断当前所处阶段，例如：
   - `draft` → 先去澄清需求
   - `clarifying` → 继续澄清需求
   - `clarified` 且未初始化任务 / 未完成任务 → 查看任务清单
   - 任务已有进度 → 继续执行任务

3. **继续上次步骤入口**
   - 首页项目卡片可增加“继续项目”入口文案优化
   - `/workspace/[id]` 页面提供一个最明确的主 CTA，直接带用户去最应该去的下一页

4. **阶段导航链接**
   工作区页至少提供这几个入口：
   - 需求澄清
   - 设计书预览
   - 任务清单

5. **基础异常处理**
   - 项目不存在
   - 项目状态与数据不一致时，给出保守兜底展示

## 实现建议

### 1. 工作区主 CTA
在 `/workspace/[id]` 顶部显著展示一个“下一步”按钮，根据项目阶段动态变化，例如：

- draft → `开始澄清需求`
- clarifying → `继续澄清需求`
- clarified 且任务未开始 → `进入任务清单`
- clarified 且任务进行中 → `继续执行任务`

跳转建议：
- 澄清相关 → `/workspace/[id]/clarify`
- 任务相关 → `/workspace/[id]/tasks`

### 2. 阶段展示
在工作区页展示一个简单阶段区块，建议包含：

- 项目创建
- 需求澄清
- 设计书预览
- 任务执行

每个阶段可显示：
- 已完成 / 当前进行中 / 未开始

无需复杂组件，保持简单清晰。

### 3. 继续项目体验
在首页最近项目卡片中，CTA 文案可更贴近当前阶段，例如：
- `继续澄清`
- `查看设计书`
- `继续任务`
- 或统一保留 `进入工作区`，但卡片上补充“下一步”提示也可以

你可以做最小实现，但要让用户知道“现在最该做什么”。
