# task1.md - 初始化项目骨架

## 任务目标
请为“零基础AI项目教练”初始化第一版 Web App 项目骨架。

本任务只做项目初始化和前两页的静态骨架，不实现数据库、登录、真实 AI 对话、真实 API。

本任务完成后，我希望得到一个可以本地运行的 Next.js 项目，包含：
1. 基础项目结构
2. 顶部导航
3. 首页静态页面
4. 路线选择页静态页面
5. 可复用的路线卡组件
6. 基础样式与布局

## 技术要求
请严格使用以下技术栈：
- Next.js（App Router）
- TypeScript
- Tailwind CSS
- shadcn/ui（如适合）

## 重要限制
请严格遵守：
1. 不要接入数据库
2. 不要接入登录系统
3. 不要实现真实 AI 对话
4. 不要实现真实 API 调用
5. 不要顺手添加与当前任务无关的复杂功能
6. 不要过度设计状态管理，先用静态数据

## 你要先做的事
在动手写代码前，请先：
1. 用简短语言总结你准备创建哪些页面、组件和文件
2. 说明项目目录结构将怎么组织
3. 说明你不会实现哪些内容
然后再开始生成代码。

## 需要实现的页面

### 页面 1：首页
首页是面向零基础用户的落地页。

需要包含：
- 顶部导航栏（Logo、工作台占位入口、登录占位按钮）
- Hero 区块
- 一句清晰的主标题
- 一段简短副标题
- 两个按钮：
  - 开始选择路线
  - 直接开始做项目
- 三个状态分流卡片：
  - 我还没装工具
  - 我已经能和模型聊天，但不会做项目
  - 我已经有工具，直接开始做产品
- 一个“你将获得什么”的成果预览模块，展示：
  - 路线选择
  - 安装引导
  - 项目设计书
  - 执行拆解
- 一个简短 FAQ 区块

页面风格要求：
- 干净
- 现代
- 有产品感
- 不花哨
- 适合 SaaS 工具首页

### 页面 2：路线选择页
路线选择页用于展示“模型 + 编程工具”的组合路线卡。

需要包含：
- 页面标题和说明文字
- 顶部简易筛选区（静态 UI，不需要真实筛选逻辑）
  - 零基础优先
  - 简单优先 / 能力优先
  - 国内优先 / 国际路线
  - IDE / 终端
- 路线卡列表

请先内置以下静态路线数据：
1. DeepSeek + OpenCode
2. 通义千问 + 通义灵码
3. 豆包 / 火山方舟 + TRAE
4. 百度千帆 / 文心 + 文心快码
5. ChatGPT + Codex CLI
6. Claude + Claude Code
7. Gemini + Gemini CLI

每张路线卡需要显示：
- 组合名称
- 一句话说明
- 适合谁
- 使用条件
- 安装难度
- 推荐标签
- 一个按钮：选择这条路线

## 组件要求
请至少抽出以下可复用组件：
- Navbar
- HeroSection
- RouteCard
- SectionHeader
- StatusCard
- FAQSection

如果你认为还需要别的简单组件，也可以加，但不要过度拆分。

## 路由要求
请至少建立以下路由：
- /
- /routes

同时为未来页面预留可扩展导航入口，但先不用实现完整页面：
- /project/new
- /workspace

如果这些页面未实现，可以先用简单占位页。

## 设计要求
整体视觉风格要求：
- 简洁
- 清晰
- 留白充足
- 文案易懂
- 不要看起来像开发者工具后台
- 更像一个面向普通人的 AI 产品

## 文案要求
请默认使用中文界面文案。
不要出现太多术语。
文案要让小白用户看得懂。

## 代码要求
- 保持代码整洁
- 组件命名清晰
- 尽量写必要注释
- 避免无意义复杂抽象
- 静态数据请单独放在一个配置文件中，而不是直接写死在页面文件里
