# task.md

## 用途
本文件不是具体开发任务。

它只负责两件事：
1. 记录任务拆分规则
2. 记录当前有哪些 task 文件

## 任务拆分规则
- 以后每个具体开发任务单独放在 task1.md、task2.md、task3.md ...
- 一个 task 只解决一个明确阶段的问题
- 不允许把多个大阶段混进同一个 task 文件
- 执行任务时，默认只执行当前指定的 task 文件

## 执行要求
开始任何 task 前，请先：
1. 总结将修改哪些文件
2. 说明本次不会实现哪些内容
3. 等确认后再大规模写代码
